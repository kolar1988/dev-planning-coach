# Dev Planning Coach

一个面向软件开发场景的教练型 skill。

它的目标不是直接写代码，而是先把需求问清楚、把现有仓库读明白、把实现路径拆出来，最后输出一份可执行的开发计划。

## 适合什么场景

当你有下面这些需求时，这个 skill 最适合：

- 先别写代码，先帮我规划
- 读一下这个项目，告诉我应该怎么改
- 把一个模糊功能拆成开发步骤
- 从零开始规划一个项目或模块
- 给我一个实现方案，而不是直接开写

## 它会怎么工作

这个 skill 的默认工作方式是：

1. 判断当前任务是“已有项目改功能”还是“从零开始做项目”
2. 如果是已有项目，先读仓库，再问问题
3. 一次只问一个高价值问题
4. 信息足够后，输出实现计划
5. 停在规划阶段，等待你决定是否进入编码

它会主动避免两类常见问题：

- 没读仓库就乱给方案
- 一次抛给你太多问题

## 输出内容

默认会整理出这些内容：

- 需求理解
- 已知事实
- 目标与非目标
- 约束与假设
- 方案选项与推荐
- 影响范围
- 分步实现计划
- 风险与验证建议

## 安装

这个仓库本身就是 skill 根目录，核心文件如下：

```text
.
├── SKILL.md
├── README.md
└── agents/
    └── openai.yaml
```

把整个目录放到你的本地 skills 目录下即可。常见放法例如：

```bash
mkdir -p ~/.agents/skills
git clone git@github.com:kolar1988/dev-planning-coach.git ~/.agents/skills/dev-planning-coach
```

如果你的运行环境使用别的 skills 目录，只要保持目录名不变，并确保 `SKILL.md` 位于 skill 根目录即可。

## 使用方式

你可以显式调用：

```text
$dev-planning-coach 帮我规划这个功能，先别写代码
```

也可以自然语言触发：

```text
读一下这个仓库，告诉我这个导出功能应该怎么改，先不要动代码
```

```text
我想做一个内部工单系统，先一步步帮我把实现方案想清楚
```

## 适用边界

这个 skill 更适合“规划优先”的任务，不适合下面这些情况：

- 范围已经很清楚，而且你就是想直接写代码
- 只是问一个很小的事实问题
- 只是做一次很明确的小 bug 修复
